# Fix Firebase Authentication Error

## Error Message
```
Firebase: Domain not allowlisted by project (auth/unauthorized-continue-uri)
```

## Solution

You need to add your Vercel deployment domain to Firebase's Authorized domains list.

### Steps to Fix:

1. **Get your Vercel deployment URL**
   - Go to your Vercel project dashboard
   - Your deployment URL will be something like: `your-project.vercel.app` or `yourproject.vercel.app`

2. **Add domain to Firebase**
   - Go to [Firebase Console](https://console.firebase.google.com)
   - Select your project: **az-genes-1ca88**
   - Navigate to: **Authentication** → **Settings** → **Authorized domains** tab
   - Click **Add domain** button
   - Enter your Vercel domain (e.g., `your-project.vercel.app`)
   - Click **Add**

3. **Add both production and preview domains** (if needed)
   - Production: `your-project.vercel.app`
   - Preview: `your-project-git-main-yourusername.vercel.app` (auto-generated by Vercel)

4. **If you have a custom domain**
   - Also add your custom domain to the Authorized domains list

### Example:
If your Vercel project is deployed at `azgenetics.vercel.app`, you would:
1. Open Firebase Console
2. Go to Authentication → Settings → Authorized domains
3. Add: `azgenetics.vercel.app`

### Why this happens:
Firebase Authentication requires that all callback URLs be from domains that are explicitly authorized. This is a security feature to prevent unauthorized redirects.

### After adding the domain:
- Your authentication should work immediately
- No code changes or re-deployment needed
- All users can now sign in/sign up properly
